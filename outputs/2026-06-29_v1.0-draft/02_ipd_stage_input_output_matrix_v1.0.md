# IPD 各阶段输入输出文件矩阵 V1.0

| 项目 | 内容 |
|---|---|
| 文档名称 | IPD 各阶段输入输出文件矩阵 |
| 版本 | V1.0 草案 |
| 适用企业 | 江苏苏威尔科技有限公司 |
| 适用范围 | 教育装备类硬件研发项目 |
| 使用方式 | 作为项目阶段启动、评审、准出和归档的执行清单 |

## 1. 使用规则

### 1.1 基本规则

- 每一阶段启动前，必须确认上一阶段准出结论。
- 每一阶段评审前，必须完成本阶段必交付文件。
- 文件可以根据项目类型裁剪，但裁剪必须在项目计划中说明，并经过 PDT 负责人或授权审批人确认。
- 所有输出文件必须有版本号、责任人、评审状态和归档位置。
- 阶段准出后，输出文件进入受控状态，后续修改必须按变更流程执行。

### 1.2 文件优先级

| 优先级 | 含义 | 管理要求 |
|---|---|---|
| P0 | 阶段准出必需文件 | 未完成不得进入下一阶段 |
| P1 | 重要支撑文件 | 原则上必须完成，特殊情况需记录豁免 |
| P2 | 项目适配文件 | 根据项目类型、产品复杂度和风险决定 |

## 9. 项目类型适配建议

V1.0 基线按完整流程执行。若后续希望提升效率，可在 V1.1 中增加下列裁剪规则。

| 项目类型 | 可保持完整的内容 | 可申请裁剪的内容 | 不可裁剪的内容 |
|---|---|---|---|
| 全新产品开发 | 全部阶段 | 无 | 需求、方案、设计、验证、发布、归档 |
| 平台型产品开发 | 计划至发布阶段 | 可复用平台可行性报告 | 需求差异、BOM、验证、发布 |
| 派生产品开发 | 计划至发布阶段 | 概念阶段可简化 | 差异需求、变更影响、回归验证 |
| 客户定制项目 | 需求至发布阶段 | 概念和计划可合并评审 | 客户需求确认、验收标准 |
| 降本优化项目 | 变更、验证、发布 | MRD / PRD 可简化 | 成本目标、替代风险、验证 |
| 器件替代项目 | 变更、验证、发布 | 概念、计划可简化 | 替代评估、样机验证、ECN |
| 质量改进项目 | 问题分析、验证、发布 | 商业分析可简化 | 根因分析、措施验证、闭环 |
| 预研项目 | 概念、计划、开发、验证 | 发布阶段可不执行 | 技术可行性、实验报告、风险结论 |

## 10. 文件模板优先级

建议按以下顺序生成模板。

| 优先级 | 模板 |
|---|---|
| 第一批 | PRD、项目计划、硬件需求规格书、硬件总体方案、原理图检查表、PCB 检查表、BOM 清单、问题跟踪表 |

| 第三批 | PVT 试产报告、生产测试方案、设计归档清单、ECR / ECO / ECN、质量改进报告 |
