# 硬件研发阶段评审点与准出标准 V1.0

| 项目 | 内容 |
|---|---|
| 文档名称 | 硬件研发阶段评审点与准出标准 |
| 版本 | V1.0 草案 |
| 适用企业 | 江苏苏威尔科技有限公司 |
| 适用范围 | 教育装备类硬件研发项目 |
| 使用方式 | 用于阶段评审、门禁判断、遗留问题管理和准出审批 |

## 1. 评审总原则

- 评审必须基于文件、数据、样机、测试记录和问题清单，不以口头说明替代正式证据。
- 评审结论必须明确为：通过、带条件通过、不通过。
- 带条件通过必须明确遗留问题、责任人、完成时间、风险接受人和验证方式。
- 涉及安全、法规、认证、重大质量、重大成本和重大交付风险的问题，不得无审批带入下一阶段。
- 阶段准出后，关键输出文件进入受控状态，后续修改必须按变更流程执行。

## 2. 问题等级定义

| 等级 | 定义 | 阶段准出要求 |
|---|---|---|
| S1 致命问题 | 影响人身安全、法规认证、核心功能、批量质量或导致项目目标无法达成 | 必须关闭，不允许带入下一阶段 |
| S2 严重问题 | 影响主要功能、性能、可靠性、生产可行性、成本或交付，但存在明确解决路径 | 原则上关闭；如带入下一阶段，必须经 PDT 负责人批准 |
| S3 一般问题 | 不影响阶段目标达成，可通过后续优化解决 | 可带入下一阶段，但必须有责任人和完成时间 |
| S4 建议项 | 优化建议或体验改进，不影响当前阶段准出 | 可记录进入后续优化池 |

## 3. 评审结论定义

| 结论 | 含义 | 后续动作 |
|---|---|---|
| 通过 | 阶段目标达成，必需输出完整，关键风险关闭或受控 | 进入下一阶段 |
| 带条件通过 | 阶段目标基本达成，存在可接受遗留问题 | 按遗留问题计划进入下一阶段 |
| 不通过 | 阶段目标未达成，关键文件缺失，或存在不可接受风险 | 整改后重新评审 |

## 4. 阶段门禁总览

| 阶段 | 评审点 | 评审目的 | 主持人 | 关键参与人 | 准出结论 |
|---|---|---|---|---|---|
| 概念阶段 | 立项 / 概念评审 | 判断项目是否值得投入 | PDT 负责人 / 产品负责人 | 产品、项目、系统、硬件、采购、质量 | 是否进入计划阶段 |
| 计划阶段 | 产品需求评审 | 确认需求完整、清晰、可验证 | 产品经理 / 项目经理 | 产品、系统、硬件、固件、测试、质量 | 是否冻结需求基线 |
| 计划阶段 | 硬件方案评审 | 确认硬件方案可实现、可验证、风险可控 | 硬件负责人 | 系统、硬件、PCB、固件、结构、测试、采购、质量 | 是否进入详细设计 |
| 开发阶段 | 原理图评审 | 确认电路设计正确性和完整性 | 硬件负责人 | 硬件、固件、测试、质量、同行专家 | 是否允许 PCB Layout |
| 开发阶段 | PCB 评审 | 确认 Layout 满足电气、结构、EMC、DFM 要求 | 硬件 / PCB 负责人 | 硬件、PCB、结构、工艺、测试、EMC | 是否允许投板 |
| 开发阶段 | BOM 评审 | 确认物料正确、可采购、成本和供应风险可控 | 硬件 / 采购 | 硬件、采购、供应链、质量、生产 | 是否允许样机采购 |
| 开发阶段 | 样机调试评审 | 确认样机达到系统验证入口条件 | 项目经理 / 硬件负责人 | 硬件、固件、测试、质量 | 是否进入 EVT |
| 验证阶段 | EVT 评审 | 确认工程样机基本功能和性能满足要求 | 测试负责人 | 硬件、固件、系统、质量、项目 | 是否进入 DVT |
| 验证阶段 | DVT 评审 | 确认设计满足需求、可靠性和专项测试要求 | 测试负责人 / 质量 | 硬件、固件、结构、可靠性、EMC、质量 | 是否设计冻结 |
| 发布阶段 | PVT 评审 | 确认生产过程、测试方案和工艺具备量产条件 | 生产 / 工艺负责人 | 生产、工艺、质量、硬件、供应链 | 是否进入量产发布 |
| 发布阶段 | 量产发布评审 | 确认产品、资料、供应、质量和生产全部准备就绪 | PDT 负责人 | 项目、产品、研发、质量、生产、供应链、售后 | 是否正式量产 |
| 生命周期阶段 | 工程变更评审 | 确认变更必要性、影响范围和验证方案 | 项目经理 / 质量 | 硬件、采购、生产、质量、售后 | 是否批准变更 |

## 5. 概念阶段评审与准出标准

### 5.1 必需输入

- 《市场需求说明书 MRD》。
- 《产品机会分析报告》。
- 《技术可行性分析报告》。
- 《初步风险清单》。
- 《初步成本估算》。

### 5.2 重点检查项

- 目标客户和应用场景是否明确。
- 产品价值、竞争优势和业务目标是否明确。
- 教育教学、考试竞赛、区域管理等场景是否已识别。
- 核心技术路径是否初步可行。
- 关键器件、供应链、认证、成本和交付风险是否初步识别。
- 项目资源是否具备基本保障。

### 5.3 准出标准

- MRD 内容完整，需求来源清楚。
- 产品机会和业务价值获得认可。
- 技术可行性无不可接受风险。
- 初步成本和资源投入在可接受范围内。
- S1 问题为 0。
- S2 问题必须有明确责任人和解决计划。
- 形成《概念阶段评审报告》。

## 6. 产品需求评审与准出标准

### 6.1 必需输入

- 《产品需求说明书 PRD》。
- 《市场需求说明书 MRD》。
- 客户需求记录。
- 竞品和历史问题输入。

### 6.2 重点检查项

- 需求是否完整覆盖目标用户、应用场景和使用流程。
- 功能需求、性能需求、接口需求、环境需求、可靠性需求是否清楚。
- 需求是否可设计、可测试、可验收。
- 需求优先级是否明确。
- 客户定制需求是否和标准产品需求区分。
- 需求变更规则是否明确。

### 6.3 准出标准

- PRD 通过评审并形成受控版本。
- 所有 P0 / P1 需求有验收标准。
- 硬件相关需求可追溯到后续硬件需求规格书。
- 需求冲突和重大不确定项已关闭或经批准保留。
- 形成《产品需求评审报告》。

## 7. 硬件方案评审与准出标准

### 7.1 必需输入

- 《产品需求说明书 PRD》。
- 《硬件需求规格书》。
- 《硬件总体方案设计说明书》。
- 《关键器件选型报告》。
- 《初版 BOM / 成本估算》。
- 《测试策略 / 验证计划》。
- 《风险管理计划》。

### 7.2 重点检查项

- 硬件架构是否满足产品需求。
- 传感器、采集、控制、通信、电源、显示、接口等方案是否合理。
- 关键器件选型是否考虑成本、交期、生命周期、替代料和供应风险。
- 硬件与固件、结构、测试、生产的接口是否明确。
- EMC、ESD、可靠性、热设计、DFM、DFT 是否有前置考虑。
- 测试验证策略是否能覆盖需求。

### 7.3 准出标准

- 硬件需求规格书和总体方案通过评审。
- 关键技术路线确定。
- 关键器件不存在不可接受供应风险。
- 主要接口定义清楚。
- 验证计划覆盖主要需求和风险。
- S1 问题为 0，S2 问题有关闭计划并获得批准。
- 允许进入详细设计。

## 8. 原理图评审与准出标准

### 8.1 必需输入

- 受控硬件需求规格书。
- 受控硬件总体方案。
- 原理图文件。
- 初版 BOM。
- 《原理图设计检查表》。

### 8.2 重点检查项

- 电源树、功耗预算、保护电路是否合理。
- MCU / 控制器 / 采集模块 / 通信接口连接是否正确。
- 复位、时钟、下载调试、启动配置是否完整。
- 传感器、执行器、接口和扩展模块是否满足应用需求。
- 防反接、过压、过流、ESD、浪涌等保护是否合理。
- 关键网络命名、测试点、调试接口是否完整。
- 关键器件封装和引脚定义是否与数据手册一致。
- 与固件、结构、测试、工艺相关约束是否明确。

### 8.3 准出标准

- 原理图检查表完成并签署。
- 关键电路经过同行评审。
- S1 / S2 电路问题关闭。
- BOM 与原理图一致。
- 所有未决问题有记录和责任人。
- 允许进入 PCB Layout。

## 9. PCB 评审与准出标准

### 9.1 必需输入

- 通过评审的原理图。
- PCB Layout 文件。
- 结构约束文件。
- 《PCB Layout 检查表》。
- 《DFM 检查表》。
- 《DFT 检查表》。

### 9.2 重点检查项

- 板框、安装孔、接口位置与结构约束一致。
- 电源、地、关键信号、时钟、通信接口布局合理。
- 高电流、电源回路、敏感模拟信号和噪声源隔离合理。
- EMC、ESD、防护、接地和屏蔽措施可实现。
- 测试点、工装定位、生产测试和维修需求已考虑。
- 丝印、极性、器件间距、焊接可制造性满足要求。
- DRC 检查无未解释错误。

### 9.3 准出标准

- PCB 评审通过。
- DRC 问题已关闭或有批准豁免。
- DFM / DFT 检查通过。
- 结构、工艺、测试关键约束已确认。
- 投板资料完整。
- 允许 PCB 投板和样机制作。

## 10. BOM 评审与准出标准

### 10.1 必需输入

- 原理图和 PCB 受控版本。
- BOM 清单。
- 关键器件选型报告。
- 器件承认资料。
- 成本估算。

### 10.2 重点检查项

- BOM 位号、规格、封装、数量、替代料是否正确。
- 关键器件是否有供应风险、停产风险或单一来源风险。
- 成本是否满足项目目标。
- 安规、环保、认证要求是否考虑。
- 采购周期是否满足项目计划。
- 物料编码、版本、状态是否清楚。

### 10.3 准出标准

- BOM 与设计文件一致。
- 关键器件承认资料完成或有明确计划。
- 高风险物料有替代策略。
- 成本偏差有说明和审批。
- 允许样机采购或试产采购。

## 11. 样机调试评审与准出标准

### 11.1 必需输入

- 样机清单和样机版本。
- 《样机调试报告》。
- 问题跟踪表。
- 设计变更记录。

### 11.2 重点检查项

- 样机硬件版本、BOM 版本和固件版本是否一致。
- 上电、基本功能、关键接口、传感器、通信、电源等是否通过。
- 已发现问题是否记录完整。
- 需要改板、飞线、替代料或参数调整的问题是否清楚。
- EVT 测试入口条件是否满足。

### 11.3 准出标准

- 样机可稳定运行主要功能。
- S1 问题为 0。
- 影响 EVT 的 S2 问题必须关闭。
- 样机、BOM、固件和调试记录版本一致。
- 允许进入 EVT。

## 12. EVT 评审与准出标准

### 12.1 必需输入

- 《EVT 测试计划》。
- 《EVT 测试报告》。
- 样机调试报告。
- 问题跟踪表。

### 12.2 重点检查项

- 基本功能是否满足 PRD 和硬件需求。
- 关键性能是否达到工程样机要求。
- 主要接口、传感器、通信和电源稳定性是否满足要求。
- 测试覆盖率是否达到计划。
- 设计问题是否分类并制定闭环计划。

### 12.3 准出标准

- EVT 计划内 P0 测试项全部完成。
- S1 问题为 0。
- 影响 DVT 的 S2 问题关闭。
- 需改版问题形成设计变更记录。
- 允许进入 DVT 或下一轮样机。

## 13. DVT 评审与准出标准

### 13.1 必需输入

- 《DVT 测试计划》。
- 《DVT 测试报告》。
- 可靠性测试报告。
- EMC / 安规测试报告或预检报告。
- 问题跟踪表。
- 设计变更记录。

### 13.2 重点检查项

- 产品功能、性能、接口、环境适应性和可靠性是否满足需求。
- EMC、ESD、安规、环保、行业认证风险是否可接受。
- 问题是否闭环，设计是否趋于稳定。
- 量产 BOM、工艺和测试方案是否具备准备条件。
- 是否具备设计冻结条件。

### 13.3 准出标准

- DVT 计划内 P0 / P1 测试项完成。
- S1 问题为 0。
- S2 问题原则上关闭；未关闭项必须经 PDT 负责人批准。
- 设计文件和 BOM 达到冻结条件。
- 允许进入 PVT。

## 14. PVT 评审与准出标准

### 14.1 必需输入

- 冻结设计资料。
- 量产 BOM。
- 《PVT 试产计划》。
- 《PVT 试产报告》。
- 《生产测试方案》。
- 工艺文件。
- 生产导入检查表。

### 14.2 重点检查项

- 试产过程是否按计划执行。
- 生产测试覆盖率、测试节拍和误判率是否满足要求。
- 工艺流程、工装夹具、作业指导书是否可执行。
- 试产良率、主要不良项和改进措施是否明确。
- 供应链、质量、生产和售后资料是否准备完成。

### 14.3 准出标准

- PVT 目标达成或偏差经批准。
- 试产重大问题关闭。
- 生产测试方案可稳定执行。
- 量产 BOM 受控发布。
- 生产、质量、供应链均确认可量产。
- 允许进入量产发布评审。

## 15. 量产发布评审与准出标准

### 15.1 必需输入

- 《量产发布评审报告》。
- 《设计归档清单》。
- 量产 BOM。
- 工艺文件。
- 生产测试方案。
- PVT 试产报告。
- 未关闭问题清单。

### 15.2 重点检查项

- 产品设计资料是否齐套归档。
- 量产 BOM、图纸、固件、工艺、测试文件是否版本一致。
- 供应链是否具备量产交付能力。
- 质量控制计划是否明确。
- 售后、维修、备件和客户交付资料是否准备完成。
- 遗留问题是否可接受。

### 15.3 准出标准

- 所有 P0 归档资料齐套。
- S1 / S2 问题关闭或经正式批准。
- 生产、质量、供应链、研发、产品共同签署发布结论。
- 产品进入生命周期阶段。

## 16. 工程变更评审与准出标准

### 16.1 必需输入

- 《工程变更申请 ECR》。
- 变更影响分析。
- 变更验证方案。
- 涉及的设计文件、BOM、工艺、测试和库存信息。

### 16.2 重点检查项

- 变更原因是否充分。
- 影响范围是否覆盖设计、BOM、库存、在制品、已出货产品、客户、认证和成本。
- 验证方案是否足以证明变更有效。
- 是否需要客户通知或认证补测。
- 是否明确切换批次、生效时间和责任人。

### 16.3 准出标准

- ECR 评审通过。
- ECO 执行计划明确。
- ECN 发布范围明确。
- 验证结果满足要求。
- 新旧版本切换和追溯规则清楚。

## 17. 评审会议记录模板字段

正式模板至少包含以下字段：

| 字段 | 说明 |
|---|---|
| 项目名称 | 项目正式名称 |
| 评审类型 | 立项、需求、方案、原理图、PCB、BOM、EVT、DVT、PVT、量产发布、工程变更等 |
| 评审日期 | 会议日期 |
| 主持人 | 评审主持人 |
| 参会人员 | 各部门参会人员 |
| 输入文件版本 | 本次评审所依据的文件版本 |
| 评审结论 | 通过 / 带条件通过 / 不通过 |
| 问题清单 | 问题编号、等级、描述、责任人、完成时间 |
| 遗留风险 | 风险描述、接受人、措施 |
| 准出意见 | 是否允许进入下一阶段 |
| 签署记录 | 相关责任人确认 |
